# 设计语言

WebOS 的视觉语言叫 **Frosted Glass**——磨砂玻璃。背景是高模糊的、微透明的，前景元素像浮在玻璃上的光。蓝色是唯一的强调色，所有注意力自然汇聚到可交互的东西上。

### 用色规则

- **蓝色** → 只有交互元素：按钮、选中项、链接、焦点、进度条
- **红色** → 只有危险操作：关闭、删除、错误
- **绿色** → 只有成功状态：保存完成、安装成功
- **黄色** → 只有警告：空间不足、权限不够
- **其他一切** → 灰度。背景深灰，文字浅灰，边框微灰

## 模糊与透明

这是整个设计语言的核心。不是随便加模糊，是有严格层次规则的。

### 层级

| 层 | 内容 | 模糊度 | 透明度 | 说明 |
|----|------|--------|--------|------|
| L0 | 桌面壁纸 | 无 | 不透明 | 壁纸本身 |
| L1 | 桌面图标 | 无 | 不透明 | 图标浮在壁纸上 |
| L2 | 窗口背景 | blur(40px) | 0.72 | 高模糊微透明 |
| L3 | 窗口内容 | 无 | 不透明 | 文字控件清晰 |
| L4 | 任务栏 | blur(40px) | 0.78 | 同窗口质感 |
| L5 | 弹出层/菜单 | blur(32px) | 0.85 | 越靠前越清晰 |
| L6 | 模态对话框 | blur(48px) | 0.90 | 最高层级对话 |
| L7 | 通知 Toast | blur(24px) | 0.82 | 轻量提示 |
| L8 | Tooltip | blur(16px) | 0.88 | 最小模糊 |

**规则**: 层级越高（越靠近用户），透明度越低，内容越清晰。所有模糊背景都能隐约透出后面的内容，制造深度感。

### 模糊实现

因为全部用 WebGPU 渲染，没有 CSS backdrop-filter，要自己做：

- **方案一**: 两遍高斯模糊（水平+垂直），compute shader 实现
- **方案二**: Kawase 模糊——多次下采样+上采样，性能更好
- **方案三**: 壁纸预生成 mipmap 链，窗口背景直接采样低分辨率 mipmap

每个窗口维护独立的 RenderTarget，最终按 Z 序合成到主画布。

## 噪点

噪点就是加质感的。让数字界面有一丝物理世界的纹理。但要极度克制。

### 哪里有噪点

| 元素 | 噪点 | 原因 |
|------|------|------|
| 窗口背景 | ✅ | 给玻璃加质感 |
| 任务栏背景 | ✅ | 跟窗口统一 |
| 弹出层/菜单背景 | ✅ | 保持一致 |
| 模态对话框背景 | ✅ | 保持一致 |
| 桌面壁纸 | ❌ | 壁纸自带视觉信息 |
| 窗口标题栏 | ❌ | 交互区，干扰按钮识别 |
| 窗口内容区 | ❌ | 要纯净保证可读 |
| 文字 | ❌ | 绝不 |
| 图标 | ❌ | 绝不 |
| 按钮 | ❌ | 交互反馈要清晰 |

**总结**: 噪点只在"背景层"出现。只要是用户要看的文字、要点的按钮、要认的图标——没有噪点。

### 噪点参数

```
类型:     Perlin Noise（不是随机点，是有机纹理感）
不透明度:  2% ~ 3%（刚好看得见，不注意就忽略）
缩放:     4x ~ 8x（大尺度纹理，不是沙粒感）
混合模式:  overlay
生成方式:  启动时预生成一张 256x256 噪点纹理上传 GPU，采样时 tile 铺开
```

### 着色器

```glsl
uniform sampler2D u_noise;        // 预生成噪点纹理
uniform float u_noise_opacity;     // 0.02 ~ 0.03
uniform float u_noise_scale;       // 4.0 ~ 8.0

vec4 apply_noise(vec4 base, vec2 uv) {
    float n = texture(u_noise, uv * u_noise_scale).r;
    vec3 noise_color = mix(vec3(0.0), vec3(1.0), n);
    return vec4(mix(base.rgb, noise_color, u_noise_opacity), base.a);
}
```

## 阴影

| 层级 | 阴影 |
|------|------|
| 窗口（聚焦） | 偏移(0,8px) 模糊32px rgba(59,130,246,0.15) 蓝色辉光 |
| 窗口（非聚焦） | 偏移(0,4px) 模糊16px rgba(0,0,0,0.3) |
| 弹出层 | 偏移(0,6px) 模糊24px rgba(0,0,0,0.4) |
| 任务栏 | 无阴影，用 1px 顶边框代替 |
| 通知 | 偏移(0,4px) 模糊16px rgba(0,0,0,0.3) |
| Tooltip | 偏移(0,2px) 模糊8px rgba(0,0,0,0.2) |

**聚焦窗口的蓝色辉光**是关键视觉信号——哪个窗口是活跃的，一眼就知道。

## 动效

| 类型 | 时长 | 缓动 | 用途 |
|------|------|------|------|
| 瞬时 | 0ms | — | 按下、开关 |
| 快速 | 120ms | ease-out | 悬停、颜色过渡 |
| 标准 | 200ms | ease-in-out | 菜单展开、Tooltip |
| 流畅 | 300ms | ease-in-out | 窗口最小化、面板展开 |
| 弹性 | 400ms | cubic-bezier(0.34,1.56,0.64,1) | 开始菜单弹出、通知滑入 |
| 减速 | 500ms | ease-out | 窗口打开/关闭 |

减少动画模式：所有时长减半，弹性缓动换 ease-in-out。

## 交互状态

每个可交互元素必须有完整的四态 + 禁用态：

| 状态 | 视觉 |
|------|------|
| Default | 基础样式 |
| Hover | 背景变亮，边框加深 |
| Active | 背景再亮，微缩放(0.98) |
| Focused | 2px 蓝色边框 + 4px 蓝色辉光 |
| Disabled | 透明度 0.38 |
