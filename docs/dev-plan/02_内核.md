# 内核

WebOS 内核用纯 C 编写，编译为 WASM 模块。提供进程管理、内存管理、调度器、IPC 和系统调用。

---

## 进程管理

### PCB（进程控制块）

```c
typedef struct {
    pid_t pid;           // 进程 ID
    uint32_t state;      // 状态: READY/RUNNING/BLOCKED/ZOMBIE
    uint32_t priority;   // 优先级: 0=LOW, 1=NORMAL, 2=HIGH, 3=REALTIME
    void* module_handle; // WASM 模块句柄
    void* stack_ptr;     // 栈指针
    void* entry_point;   // 入口点
    uint32_t flags;      // 标志位
    int32_t exit_code;   // 退出码
} process_t;
```

### 状态机

```
CREATED → READY ⇄ RUNNING → ZOMBIE
              ↓        ↑
            BLOCKED ────┘
```

- **READY**: 就绪，等待调度
- **RUNNING**: 正在执行
- **BLOCKED**: 等待 IPC / I/O
- **ZOMBIE**: 已退出，等待父进程 wait()

### 进程 API

| 函数 | 说明 |
|------|------|
| `process_spawn(module, priority)` | 创建新进程 |
| `process_kill(pid)` | 终止进程 |
| `process_exit(code)` | 退出当前进程 |
| `process_wait(pid, &exit_code)` | 等待子进程 |
| `process_getpid()` | 获取当前 PID |
| `process_get(pid)` | 获取 PCB |
| `process_current()` | 获取当前进程 PCB |

最大进程数: **64**

## 内存管理

### 页帧分配

- 页大小: **64KB**（WASM 页）
- 总页数: 256（16MB 默认）
- 内核保留: 4 页（256KB）

### 内存区域类型

| 类型 | 说明 |
|------|------|
| `MEM_REGION_FREE` | 未分配 |
| `MEM_REGION_USED` | 已分配 |
| `MEM_REGION_KERNEL` | 内核保留 |
| `MEM_REGION_SHARED` | 共享内存 |

### 分配器

当前使用 **bump allocator**（简单递增分配），配合区域跟踪表。最大跟踪 256 个区域。

```c
void* memory_alloc(uint32_t pages);       // 分配页
int32_t memory_free(void* ptr);           // 释放
void* memory_map_shared(uint32_t size, uint32_t owner_pid);  // 共享内存
void memory_stats(mem_stats_t* stats);    // 统计信息
```

### kmalloc

内核小对象分配用链表式 free-list allocator，在内核保留区域内运作。

## 调度器

### 算法: Round-Robin + 优先级

- 基础算法: 轮转调度（Round-Robin）
- 优先级提升: REALTIME 进程优先调度
- 时间片: 100ms
- 协作式调度: 进程主动 yield

### API

| 函数 | 说明 |
|------|------|
| `scheduler_run()` | 启动调度循环 |
| `scheduler_yield()` | 让出当前时间片 |
| `scheduler_block()` | 阻塞当前进程 |
| `scheduler_unblock(pid)` | 解除阻塞 |

## IPC（进程间通信）

### 消息队列

每个进程维护一个消息通道:

```c
typedef struct {
    pid_t sender;
    pid_t receiver;
    uint32_t type;
    uint32_t length;
    uint8_t data[4096];  // 最大 4KB
} ipc_msg_t;
```

- 每进程最多 **32** 条待处理消息
- 单条消息最大 **4096** 字节

### 消息类型

| 类型 | 值 | 说明 |
|------|-----|------|
| `IPC_MSG_SYSCALL` | 0x01 | 系统调用请求 |
| `IPC_MSG_SIGNAL` | 0x02 | 信号通知 |
| `IPC_MSG_DATA` | 0x03 | 数据传输 |
| `IPC_MSG_REQUEST` | 0x04 | 服务请求 |
| `IPC_MSG_RESPONSE` | 0x05 | 服务响应 |
| `IPC_MSG_EVENT` | 0x06 | 系统事件 |

### 通信方式

| 方式 | API | 说明 |
|------|-----|------|
| 异步消息 | `ipc_send()` / `ipc_recv()` | 非阻塞发送，可阻塞接收 |
| 同步调用 | `ipc_call()` / `ipc_reply()` | 发送后等待响应 |
| 共享内存 | `memory_map_shared()` | 高速数据交换 |
| 信号 | 通过 `IPC_MSG_SIGNAL` | 轻量通知 |
| 管道 | 基于 IPC 消息实现 | 字节流 |

## 系统调用

### 调用流程

```
用户空间 → kernel_syscall() → syscall_dispatch() → handler → host_syscall_invoke() → RuntimeBridge → 浏览器 API
```

### 系统调用号范围

| 范围 | 类别 |
|------|------|
| 0x0100-0x01FF | 文件系统 |
| 0x0200-0x02FF | 进程管理 |
| 0x0300-0x03FF | 内存管理 |
| 0x0400-0x04FF | 时间 |
| 0x0500-0x05FF | I/O |
| 0x0600-0x06FF | 网络 |
| 0x0700-0x07FF | 调试 |
| 0x0800-0x08FF | IPC |
| 0x0900-0x09FF | 通知 |
| 0x0A00-0x0AFF | 动态链接 |

### 核心 16 个系统调用

| 号 | 名称 | 参数 | 说明 |
|----|------|------|------|
| 0x01 | SYS_PRINT | ptr, len | 输出日志 |
| 0x0100 | SYS_FS_OPEN | path, flags | 打开文件 |
| 0x0101 | SYS_FS_READ | fd, buf, count | 读文件 |
| 0x0102 | SYS_FS_WRITE | fd, buf, count | 写文件 |
| 0x0103 | SYS_FS_CLOSE | fd | 关闭文件 |
| 0x0200 | SYS_PROC_EXIT | code | 退出进程 |
| 0x0201 | SYS_PROC_GETPID | — | 获取 PID |
| 0x0202 | SYS_PROC_SPAWN | module, priority | 创建进程 |
| 0x0300 | SYS_MEM_ALLOC | pages | 分配内存 |
| 0x0301 | SYS_MEM_FREE | ptr | 释放内存 |
| 0x0800 | SYS_IPC_SEND | target, type, data, len | 发送消息 |
| 0x0801 | SYS_IPC_RECV | buf, timeout | 接收消息 |
| 0x0A00 | SYS_DYNLINK_LOAD | name, len | 加载模块 |
| 0x0A01 | SYS_DYNLINK_UNLOAD | handle | 卸载模块 |
| 0x0A02 | SYS_DYNLINK_SYM | handle, sym, len | 解析符号 |
| 0x0900 | SYS_NOTIFY | title, body, type | 发送通知 |

### 动态链接器

加载和管理 .Wdll 共享模块:

- 最大 **32** 个同时加载的模块
- 引用计数，自动卸载
- 符号解析通过宿主桥接
