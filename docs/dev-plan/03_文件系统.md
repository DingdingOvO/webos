# 文件系统

WebFS 是 WebOS 的真实文件系统，不是模拟。它有 superblock、inode、数据块，完整的磁盘布局——只是后端存储在 IndexedDB 而不是物理磁盘。

---

## 磁盘布局

```
┌─────────────┐  Block 0
│  Superblock  │  4KB
├─────────────┤  Block 1
│  Block Bitmap│  可变
├─────────────┤
│  Inode Bitmap│  可变
├─────────────┤
│  Inode Table │  可变
├─────────────┤
│  Data Blocks │  可变
└─────────────┘
```

### 基本参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 块大小 | 4KB | 数据块和 inode 表的基本单位 |
| Magic | `0x57454246` ("WEBF") | Superblock 魔数 |
| Inode 大小 | 128 字节 | 紧凑的 inode 结构 |
| 最大文件 | ~4GB | 12 直接 + 1 间接 + 1 二级间接 |

---

## Superblock

```c
#define WEBFS_MAGIC 0x57454246  // "WEBF"

typedef struct {
    uint32_t magic;           // 0x57454246
    uint32_t version;         // 1
    uint32_t block_size;      // 4096
    uint32_t total_blocks;    // 总块数
    uint32_t free_blocks;     // 空闲块数
    uint32_t inode_count;     // inode 总数
    uint32_t free_inodes;     // 空闲 inode 数
    uint32_t inode_table_start; // Inode 表起始块号
    uint32_t data_blocks_start; // 数据区起始块号
    uint32_t bitmap_start;    // 位图起始块号
    uint32_t mount_count;     // 挂载次数
    uint32_t last_mount_time; // 最后挂载时间
} superblock_t;
```

---

## Inode（128 字节）

```c
typedef struct {
    uint32_t mode;            // 文件类型 + 权限
    uint32_t uid;             // 所有者
    uint32_t size;            // 文件大小（字节）
    uint32_t atime;           // 访问时间
    uint32_t mtime;           // 修改时间
    uint32_t ctime;           // 创建时间
    uint32_t nlinks;          // 硬链接数
    uint32_t blocks;          // 占用块数
    uint32_t direct[12];      // 直接块指针 (12 × 4KB = 48KB)
    uint32_t indirect;        // 一级间接 (1024 × 4KB = 4MB)
    uint32_t double_indirect; // 二级间接 (1024² × 4KB ≈ 4GB)
    uint8_t  reserved[16];    // 保留
} inode_t;                    // 总计 128 字节
```

### 寻址能力

| 层级 | 指针数 | 容量 |
|------|--------|------|
| 直接 | 12 | 48KB |
| 一级间接 | 1024 | 4MB |
| 二级间接 | 1024 × 1024 | ~4GB |
| **总计** | | **~4GB** |

## 目录项

```c
typedef struct {
    uint32_t inode;           // Inode 号
    uint16_t name_len;        // 文件名长度
    uint8_t  type;            // 文件类型 (0=普通, 1=目录)
    char     name[252];       // 文件名
} dir_entry_t;                // 256 字节
```

## VFS 层 API

```c
// 打开/关闭
int  fs_open(const char* path, int flags);
int  fs_close(int fd);

// 读写
int  fs_read(int fd, void* buf, int count);
int  fs_write(int fd, const void* buf, int count);

// 定位
int  fs_seek(int fd, int offset, int whence);

// 目录操作
int  fs_mkdir(const char* path);
int  fs_rmdir(const char* path);
int  fs_listdir(const char* path, dir_entry_t* entries, int max);
int  fs_chdir(const char* path);

// 文件操作
int  fs_unlink(const char* path);
int  fs_rename(const char* old_path, const char* new_path);
int  fs_stat(const char* path, struct stat* buf);

// 挂载
int  fs_mount(const char* device, const char* mountpoint, const char* fs_type);
int  fs_unmount(const char* mountpoint);
```

### 打开标志

| 标志 | 值 | 说明 |
|------|-----|------|
| `O_RDONLY` | 0 | 只读 |
| `O_WRONLY` | 1 | 只写 |
| `O_RDWR` | 2 | 读写 |
| `O_CREAT` | 4 | 不存在则创建 |
| `O_TRUNC` | 8 | 截断为 0 |
| `O_APPEND` | 16 | 追加写入 |

### 文件描述符

- 最大 **64** 个同时打开的文件描述符
- 0-2 预留: stdin, stdout, stderr

## 存储后端

WebFS 的磁盘存储在 **IndexedDB** 中:

- 每个块存为一条记录 (block_number → ArrayBuffer)
- Superblock 单独存储
- 异步读写通过 Promise 桥接到 WASM

### IndexedDB 结构

```
Database: "webos-fs"
  Object Store: "blocks"
    Key: block_number (uint32)
    Value: ArrayBuffer (4096 bytes)
  Object Store: "superblock"
    Key: 0
    Value: superblock_t (serialized)
```
