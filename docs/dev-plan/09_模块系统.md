# 模块系统

WebOS 的模块系统定义了三种模块类型，通过 WASM 自定义段（custom section）标识。

---

## 模块类型

### .wex — 可执行模块

应用和可独立运行的程序。有 `main()` 入口函数。

| 属性 | 值 |
|------|-----|
| 自定义段名 | `module_type` |
| 段内容 | `wex` |
| 入口 | `_main` |
| 编译器 | `em++` (C++) |
| 示例 | `calculator.wex`, `terminal.wex` |

### .Wdll — 动态库

可被内核动态加载和链接的共享库。服务、驱动都是 .Wdll。

| 属性 | 值 |
|------|-----|
| 自定义段名 | `module_type` |
| 段内容 | `Wdll` |
| 导出 | `EMSCRIPTEN_KEEPALIVE` 标记的函数 |
| 编译器 | `emcc` (C) 或 `em++` (C++) |
| 示例 | `gpu_driver.Wdll`, `audio_server.Wdll` |

### .wli — 库接口描述

JSON 格式的接口描述文件，由工具自动生成。描述 .Wdll 导出的函数签名。

| 属性 | 值 |
|------|-----|
| 格式 | JSON |
| 生成工具 | `tools/gen_wli.py` |
| 用途 | 编译时类型检查、文档生成 |

### .wli 示例

```json
{
  "module": "audio_server",
  "type": "Wdll",
  "version": "0.0.1beta",
  "exports": [
    {
      "name": "audio_open",
      "params": ["i32", "i32"],
      "return": "i32"
    },
    {
      "name": "audio_close",
      "params": ["i32"],
      "return": "i32"
    },
    {
      "name": "audio_write",
      "params": ["i32", "i32", "i32"],
      "return": "i32"
    }
  ]
}
```

## 动态加载流程

```
1. kernel → dynlink_load("audio_server.Wdll")
2.         → host_syscall_invoke(SYS_DYNLINK_LOAD, ...)
3.         → DynamicLoader.loadModule("audio_server.Wdll")
4.         → fetch("/wasm/audio_server.Wdll")
5.         → WebAssembly.compile(bytes)
6.         → 读取 module_type 自定义段 → "Wdll"
7.         → WebAssembly.instantiate(module, importObject)
8.         → 返回 module_handle
9. kernel → dynlink_resolve(handle, "audio_open", ...)
10.        → 获取导出函数指针
11.        → 后续通过函数指针调用
```

## 自定义段编码

WASM 自定义段格式:

```
0x00              Section ID (custom section = 0)
LEB128(size)      段大小
LEB128(name_len)  段名长度
"module_type"     段名 (UTF-8)
"wex" / "Wdll"   段内容 (UTF-8)
```

### 工具: add_module_section.py

编译后的 WASM 文件需要添加自定义段:

```bash
python3 tools/add_module_section.py calculator.wasm wex
python3 tools/add_module_section.py gpu_driver.wasm Wdll
```

### 工具: gen_wli.py

从 .Wdll 自动生成 .wli 接口描述:

```bash
python3 tools/gen_wli.py audio_server.Wdll > audio_server.wli
```

## 工具链

### 编译命令

**内核/引导加载器**（纯 WASM，不用 Emscripten）:

```bash
clang -target wasm32 -nostdlib -Wl,--no-entry \
      -Wl,--export=main \
      -o kernel.wasm main.c memory.c process.c ...
python3 tools/add_module_section.py kernel.wasm wex
```

**驱动/服务/应用**（Emscripten，需要浏览器环境支持）:

```bash
# .Wdll
emcc -s WASM=1 -s SIDE_MODULE=1 -O2 \
     -s EXPORTED_FUNCTIONS='["_init","_shutdown"]' \
     -o gpu_driver.Wdll gpu_driver.c
python3 tools/add_module_section.py gpu_driver.Wdll Wdll

# .wex
em++ -s WASM=1 -s EXPORTED_FUNCTIONS='["_main"]' \
     -s EXPORTED_RUNTIME_METHODS='["ccall","cwrap"]' \
     -s ENVIRONMENT='web' -s ALLOW_MEMORY_GROWTH=1 \
     -O2 -o calculator.wex main.cpp
python3 tools/add_module_section.py calculator.wex wex
```

## 运行时桥接 (importObject)

WASM 模块实例化时，宿主提供 `importObject`:

```typescript
const importObject = {
  env: {
    memory: sharedMemory,        // 共享线性内存
  },
  webos: {
    syscall_invoke: (num, a1, a2, a3, a4, a5) => {
      return bridge.dispatch(num, a1, a2, a3, a4, a5);
    },
    js_console_log: (ptr, len) => { ... },
    js_time_now: () => performance.now(),
    js_storage_read: (block, buf) => { ... },
    js_storage_write: (block, buf) => { ... },
    js_input_get_events: (buf, max) => { ... },
    js_gpu_create_context: () => { ... },
    js_gpu_draw: (pipeline, vertices) => { ... },
    js_audio_create_context: () => { ... },
    js_audio_play_pcm: (ptr, len, rate) => { ... },
    js_audio_set_volume: (ctx, vol) => { ... },
  },
};
```

所有 WASM → 浏览器的调用都通过这个桥接，WASM 模块永远不直接调用浏览器 API。
